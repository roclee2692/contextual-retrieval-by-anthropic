# 数据说明

## 数据来源

**名称**：华北水利水电大学龙子湖校区餐饮档口菜品价格表  
**格式**：PDF  
**大小**：约270,000字符  
**获取方式**：公开信息（食堂菜单）

## 数据结构

### 餐厅分布
- 一号餐厅（Canteen 1）
- 二号餐厅（Canteen 2）
- 民族餐厅（Ethnic Restaurant）

### 数据格式示例

```
[NCWU Longzihu | 一号餐厅 | 一楼 | Window 42]
五、天津包子（21号）
2元品类
- 招牌鲜肉包、辣子鸡丁包、梅干菜肉包
3元品类
- 虾仁包、叉烧包、蟹黄包
```

### 数据特点

1. **结构化列表**：档口名 - 菜品 - 价格
2. **缺少自然语境**：无情感词、无修辞、无因果关系
3. **实体密集**：80+档口，2000+菜品

## 数据统计

| 维度 | 数量 |
|-----|------|
| 餐厅 | 3个 |
| 档口 | 80+ |
| 菜品 | 2000+ |
| 价格范围 | 1.5元 - 30元 |

## 隐私处理

- ✅ 数据为公开信息（食堂菜单）
- ✅ 无个人敏感信息
- ✅ 无需脱敏处理
- ✅ 可直接用于研究

## 使用说明

### 数据放置
将PDF文件放入 `data/` 目录：
```
data/
  └── 龙子湖校区餐饮档口菜品价格表.pdf
```

### 数据处理
运行数据库创建脚本：
```bash
python create_save_db.py
```

生成的索引文件：
- `src/db/canteen_db_vectordb/` - ChromaDB向量数据库
- `src/db/canteen_db_bm25/` - BM25索引

## 数据局限性

### 为什么这个数据集不适合RAG？

**根本问题**：数据是**结构化列表**，缺少自然语言语境

**对比**：

| 特征 | 食堂菜单 | 自然文本（如影评） |
|-----|---------|------------------|
| 格式 | 列表 | 段落 |
| 语境 | ❌ 无 | ✅ 有 |
| 情感 | ❌ 无 | ✅ 有 |
| 逻辑关系 | ❌ 无 | ✅ 有 |
| 适合CR | ❌ 否 | ✅ 是 |

### 适合的数据类型

推荐用于RAG研究的数据：
1. **电影评论**（豆瓣、IMDb）
2. **知乎问答**
3. **新闻文章**
4. **学术论文**
5. **技术文档**

## 替代数据集推荐

如需使用标准数据集，推荐：

- **DuReader**：百度阅读理解数据集
- **CMRC 2018**：中文机器阅读理解
- **C-MTEB**：中文文本嵌入基准
